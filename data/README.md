# Data

## `raw/housing_training.csv`

A **synthetic** dataset (2,500 rows) generated by [`ml/generate_data.py`](../ml/generate_data.py).
Price is built from an interpretable linear formula plus noise, so a trained
linear model recovers coefficients close to the ground truth below — which
is what makes the itemized price calculation in the UI meaningful.

```
price = 45,000                       (base price)
      + 118      × area_sqft
      + 9,000    × bedrooms
      + 6,500    × bathrooms
      - 950      × age_years
      - 2,100    × distance_km
      + 650      × floor_number
      + 11,000   × has_parking
      + 8,500    × has_garden
      + 14,000   × locality_score
      + noise
```

To regenerate it (e.g. with a different seed or sample size):

```bash
python ml/generate_data.py
```

### Columns

| Column            | Type  | Description                                    | Typical range |
|--------------------|-------|--------------------------------------------------|----------------|
| `area_sqft`         | float | Living area                                       | 250 – 5,500    |
| `bedrooms`           | int   | Number of bedrooms                                | 1 – 5          |
| `bathrooms`          | int   | Number of bathrooms                               | 1 – 5          |
| `age_years`          | float | Age of the property                               | 0 – 75         |
| `distance_km`        | float | Distance to city center                           | 0.2 – 35       |
| `floor_number`       | int   | Floor the unit is on (0 = ground)                 | 0 – 19         |
| `has_parking`        | int   | 1 if a dedicated parking spot is included          | 0 or 1         |
| `has_garden`         | int   | 1 if a private garden/yard is included             | 0 or 1         |
| `locality_score`     | float | Composite amenity/safety/school-quality index      | 0 – 10         |
| `price`              | float | Sale price in USD (target)                         | —              |

## `locality/locality_houses.json`

Generated by [`ml/generate_locality.py`](../ml/generate_locality.py). Defines
the small fictional "Maple Grove" locality shown on the map: 24 houses
across 3 streets, each with map coordinates (`x`, `y`) for SVG placement and
a feature set used to compute its live price prediction when clicked in the
UI. Regenerate with:

```bash
python ml/generate_locality.py
```

### Using real data instead

Replace `raw/housing_training.csv` with any dataset that has the same 9
feature columns + a `price` target (e.g. a public real-estate dataset for
your region), then re-run:

```bash
python ml/train.py
```

Note that the "calculation" breakdown in the UI assumes a **linear**
relationship between features and price. If you swap in real data with
non-linear pricing effects, consider adding polynomial/interaction terms
or presenting the breakdown as an approximation.
